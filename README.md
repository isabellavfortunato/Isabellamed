# Rotina de medicação

Este repositório guarda uma página simples para acompanhar a rotina diária de medicação e suplemento, com marcação por dia e visão do mês inteiro.

## O que é cada arquivo

O arquivo index.html é a página inteira, tudo está dentro dele. É só esse arquivo que importa. O GitHub Pages publica ele num endereço da internet para você abrir no celular ou no computador. Mantenha o nome index.html, porque o GitHub Pages abre esse arquivo por padrão.

## Como publicar no GitHub Pages

1. Entre em github.com e faça login. Se ainda não tiver conta, crie uma, é gratuito.
2. Clique em New para criar um repositório. Dê um nome, por exemplo rotina, e deixe como público. Clique em Create repository.
3. Dentro do repositório, clique em Add file e depois em Upload files. Arraste o arquivo index.html para lá, mantendo esse nome exato. Clique em Commit changes.
4. Vá em Settings e depois em Pages, no menu lateral.
5. Em Branch, escolha main e a pasta root. Clique em Save.
6. Espere um ou dois minutos. O endereço da página aparece no topo dessa mesma tela, algo como https://seuusuario.github.io/rotina.
7. Abra esse endereço no iPhone. Se quiser, use a opção compartilhar e adicionar à tela de início, para ficar como um atalho de aplicativo.

## Como atualizar depois

Sempre que a gente mudar alguma coisa, o único arquivo que muda é o index.html. Para atualizar, entre no repositório, clique em Add file e Upload files, suba o index.html novo por cima do antigo e clique em Commit changes. Em um ou dois minutos a página atualiza sozinha.

## Sobre as marcações salvas

As marcações ficam guardadas no próprio aparelho e navegador onde você abrir. O histórico do iPhone e o do Mac ficam separados entre si. Se você limpar os dados do navegador daquele aparelho, o histórico dele é apagado.
